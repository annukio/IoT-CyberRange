IT/OT Convergence

Why Industrial Networks Are Different

Operational Technology (OT) networks control physical processes: conveyor belts, pressure valves, PLCs. Unlike IT systems, a failure here can cause physical damage or safety incidents. Historically, OT networks were completely isolated from corporate IT (also called air-gapped).

As factories modernized (Industry 4.0), IT and OT became interconnected for efficiency. This eliminated the air gap and introduced new attack surfaces without updating security practices.

NIST SP 800-82 - ICS Security Guide

Network Segmentation Recommendations

NIST recommends a Defense-in-Depth model with clearly defined zones:

  • Enterprise Zone (Corporate IT)
  • DMZ (Demilitarized Zone) - controlled buffer between IT and OT
  • Manufacturing Zone (OT/ICS)
  • Field Devices (PLCs, sensors)

Traffic between zones should be explicitly allowed by policy, with all other traffic blocked by default. This is the Allowlist model.

MITRE ATT&CK for ICS - T0822

Lateral Movement via Network Pivoting

In a flat network, an attacker who compromises the Corporate WS can pivot directly to OT assets. MITRE documents this under T0822 - Exploitation of Remote Services in the ICS matrix.

A properly segmented network forces an attacker to cross a controlled boundary - creating detection opportunities and limiting attack surface.

COMMAND: iptables Fundamentals

How Linux Packet Filtering Works

iptables controls packets at the kernel level using chains of rules. For routing between networks, the FORWARD chain applies - it processes packets that pass through the host.

A safe default policy is DROP - block everything, then explicitly allow what is needed:

iptables -P FORWARD DROP
iptables -A FORWARD -s 172.20.0.0/24 -d 172.22.0.0/24 -j DROP
iptables -A FORWARD -m state --state ESTABLISHED,RELATED -j ACCEPT

The ESTABLISHED,RELATED rule allows return traffic for connections that were already permitted - essential for TCP to function correctly.
